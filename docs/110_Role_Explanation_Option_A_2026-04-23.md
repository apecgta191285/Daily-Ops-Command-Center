# Role Explanation for Option A Oral Defense
วันที่: 23 เมษายน 2026

## 1) Admin
ความหมายในโลกจริง:
- `อาจารย์ผู้รับผิดชอบ`
- `ผู้ได้รับมอบหมายดูแลระบบ`

หน้าที่ที่ควรพูด:
- ดูแลความถูกต้องของ template ที่ใช้ตรวจห้อง
- ดูแลบัญชีผู้ใช้ของทีม
- ดูภาพรวม dashboard และติดตาม incident ได้

สิ่งที่ไม่ควรพูดเกินจริง:
- ไม่ใช่ super admin ของหลายองค์กร
- ไม่ใช่ owner ของ enterprise platform

## 2) Supervisor
ความหมายในโลกจริง:
- `lab boy`
- `เจ้าหน้าที่แล็บ`
- `ผู้ดูแลห้อง`

หน้าที่ที่ควรพูด:
- ดูว่าวันนี้ห้องไหนมีปัญหา
- เปิด dashboard, queue, detail, history เพื่อติดตามงาน
- รับช่วงต่อจาก incident ที่นักศึกษาแจ้ง

สิ่งที่ไม่ควรพูดเกินจริง:
- ไม่ใช่ manager หลายสาขา
- ไม่ใช่ compliance officer

## 3) Staff
ความหมายในโลกจริง:
- `นักศึกษาที่เข้าเวรตรวจห้องตามรอบ`

หน้าที่ที่ควรพูด:
- เลือกห้องที่ตัวเองรับผิดชอบ
- ทำ checklist ตามช่วงเวลา
- รายงาน incident เมื่อพบปัญหา

สิ่งที่ไม่ควรพูดเกินจริง:
- ไม่ใช่ช่างซ่อมเต็มรูปแบบ
- ไม่ใช่ผู้บริหารจัดการ asset inventory

## 4) One-Sentence Mapping
ใช้ประโยคนี้เวลาอยากสรุปเร็ว:

> Admin คืออาจารย์ผู้รับผิดชอบ, Supervisor คือเจ้าหน้าที่หรือผู้ดูแลห้อง, และ Staff คือนักศึกษาที่เข้าเวรตรวจห้องตามรอบ

## 5) What This Role Model Helps Defend
- อธิบายได้ว่าทำไมระบบนี้เป็น internal-only
- อธิบายได้ว่าทำไมมีแค่ 3 roles
- อธิบายได้ว่าทำไม checklist, incident, dashboard, history, และ governance อยู่ในระบบเดียวกัน
