# Process Description Draft v1

## Process 1.0 : จัดการผู้ใช้และสิทธิ์

| Field       | Detail                                                                    |
| ----------- | ------------------------------------------------------------------------- |
| Process     | 1.0 จัดการผู้ใช้และสิทธิ์ |
| Description | กระบวนการสำหรับผู้ดูแลระบบในการจัดการบัญชีผู้ใช้ บทบาท สิทธิ์การใช้งาน และสถานะการเปิดใช้งานของผู้ใช้ภายในระบบ A-lite Daily Ops Command Center |
| Input       | ข้อมูลจัดการผู้ใช้จาก Admin; ข้อมูลผู้ใช้และสิทธิ์จาก D1 ข้อมูลผู้ใช้ |
| Output      | ผลการจัดการผู้ใช้กลับไปยัง Admin; ข้อมูลผู้ใช้ที่ถูกบันทึกหรือปรับปรุงใน D1 ข้อมูลผู้ใช้ |
| Data Store  | D1 ข้อมูลผู้ใช้ (`users`) |
| Actor       | Admin |
| Task        | 1. รับข้อมูลการจัดการผู้ใช้จาก Admin<br>2. ตรวจสอบข้อมูลบัญชีผู้ใช้ บทบาท และสถานะการใช้งานที่เกี่ยวข้อง<br>3. บันทึกหรือปรับปรุงข้อมูลผู้ใช้ใน D1 ข้อมูลผู้ใช้<br>4. ส่งผลการจัดการผู้ใช้กลับให้ Admin |
| Notes       | กระบวนการนี้จำกัดเฉพาะการจัดการผู้ใช้ภายในระบบ ไม่รวม public signup หรือการลงทะเบียนผู้ใช้สาธารณะ |

## Process 2.0 : จัดการแม่แบบรายการตรวจ

| Field       | Detail                                                                    |
| ----------- | ------------------------------------------------------------------------- |
| Process     | 2.0 จัดการแม่แบบรายการตรวจ |
| Description | กระบวนการสำหรับผู้ดูแลระบบในการจัดการแม่แบบ checklist และรายการตรวจที่ใช้เป็นต้นแบบสำหรับการตรวจเช็กประจำวัน |
| Input       | ข้อมูลแม่แบบรายการตรวจจาก Admin; ข้อมูลแม่แบบและรายการตรวจเดิมจาก D3 ข้อมูลแม่แบบรายการตรวจ |
| Output      | ผลการจัดการแม่แบบกลับไปยัง Admin; ข้อมูลแม่แบบ checklist และ checklist items ที่ถูกบันทึกหรือปรับปรุงใน D3 ข้อมูลแม่แบบรายการตรวจ |
| Data Store  | D3 ข้อมูลแม่แบบรายการตรวจ (`checklist_templates`, `checklist_items`) |
| Actor       | Admin |
| Task        | 1. รับข้อมูลการสร้างหรือปรับปรุงแม่แบบ checklist จาก Admin<br>2. จัดการรายละเอียดแม่แบบ เช่น ชื่อ รายละเอียด scope และสถานะการใช้งาน<br>3. จัดการรายการตรวจภายใต้แม่แบบ เช่น ชื่อรายการ กลุ่มรายการ ลำดับ และสถานะรายการบังคับ<br>4. บันทึกหรือปรับปรุงข้อมูลใน D3 ข้อมูลแม่แบบรายการตรวจ<br>5. ส่งผลการจัดการแม่แบบกลับให้ Admin |
| Notes       | กระบวนการนี้จัดการแม่แบบและรายการตรวจเท่านั้น ไม่รวมการสร้างหรือดำเนินการ checklist run ประจำวัน |

## Process 3.0 : ดำเนินการตรวจเช็กประจำวัน

| Field       | Detail                                                                    |
| ----------- | ------------------------------------------------------------------------- |
| Process     | 3.0 ดำเนินการตรวจเช็กประจำวัน |
| Description | กระบวนการที่ Staff ใช้ดำเนินการตรวจเช็กประจำวันตามห้องและช่วงงานที่เลือก โดยระบบตรวจสอบผู้ใช้ ห้อง แม่แบบ checklist ที่พร้อมใช้งาน และจัดการรอบการตรวจของวัน |
| Input       | ข้อมูลเข้าสู่ระบบจาก Staff; ข้อมูลห้องและช่วงงานจาก Staff; ข้อมูลผลการตรวจเช็กจาก Staff; ข้อมูลผู้ใช้และสิทธิ์จาก D1; ข้อมูลห้องจาก D2; ข้อมูลแม่แบบรายการตรวจจาก D3; สถานะ checklist ของวันจาก D4 |
| Output      | รายการตรวจเช็กและผลการบันทึกกลับไปยัง Staff; ข้อมูล checklist run และผลการตรวจเช็กที่บันทึกใน D4 ข้อมูลการตรวจเช็กประจำวัน |
| Data Store  | D1 ข้อมูลผู้ใช้ (`users`); D2 ข้อมูลห้อง (`rooms`); D3 ข้อมูลแม่แบบรายการตรวจ (`checklist_templates`, `checklist_items`); D4 ข้อมูลการตรวจเช็กประจำวัน (`checklist_runs`, `checklist_run_items`) |
| Actor       | Staff |
| Task        | 1. ตรวจสอบข้อมูลเข้าสู่ระบบและสิทธิ์ของผู้ปฏิบัติงานจาก D1<br>2. รับข้อมูลห้องและช่วงงานที่ Staff เลือก และตรวจสอบข้อมูลห้องจาก D2<br>3. ค้นหาแม่แบบ checklist ที่พร้อมใช้งานจาก D3 ตามห้องและช่วงงานที่เกี่ยวข้อง<br>4. ตรวจสอบสถานะ checklist ของวันจาก D4 และสร้างหรือใช้ checklist run ที่มีอยู่ตามเงื่อนไขของระบบ<br>5. บันทึกผลการตรวจเช็กของรายการ checklist ลงใน D4<br>6. ส่งรายการตรวจเช็ก สถานะของวัน และผลการบันทึกกลับให้ Staff |
| Notes       | กระบวนการนี้ไม่รวมการจัดการสถานะ incident หรือการติดตาม dashboard/history |

## Process 4.0 : จัดการเหตุผิดปกติ

| Field       | Detail                                                                    |
| ----------- | ------------------------------------------------------------------------- |
| Process     | 4.0 จัดการเหตุผิดปกติ |
| Description | กระบวนการสำหรับบันทึกเหตุผิดปกติจาก Staff และปรับปรุงสถานะหรือข้อมูลการติดตามโดย Supervisor หรือ Admin พร้อมบันทึกกิจกรรมของเหตุผิดปกติเป็นประวัติการทำงานของระบบ |
| Input       | ข้อมูลเหตุผิดปกติจาก Staff; ข้อมูลอัปเดตสถานะเหตุผิดปกติจาก Supervisor หรือ Admin; ข้อมูลห้องจาก D2; ข้อมูลเหตุผิดปกติเดิมจาก D5 |
| Output      | ผลการบันทึกเหตุผิดปกติกลับไปยัง Staff; ผลการอัปเดตสถานะกลับไปยัง Supervisor หรือ Admin; ข้อมูลเหตุผิดปกติและสถานะที่บันทึกหรือปรับปรุงใน D5; ข้อมูลกิจกรรมเหตุผิดปกติที่บันทึกใน D6 |
| Data Store  | D2 ข้อมูลห้อง (`rooms`); D5 ข้อมูลเหตุผิดปกติ (`incidents`); D6 ข้อมูลกิจกรรมเหตุผิดปกติ (`incident_activities`) |
| Actor       | Staff, Supervisor, Admin |
| Task        | 1. รับข้อมูลเหตุผิดปกติจาก Staff<br>2. ตรวจสอบข้อมูลห้องที่เกี่ยวข้องจาก D2 และตรวจสอบรายละเอียดเหตุผิดปกติ<br>3. บันทึกรายงานเหตุผิดปกติลงใน D5<br>4. บันทึกกิจกรรมการสร้างเหตุผิดปกติลงใน D6<br>5. รับข้อมูลอัปเดตสถานะหรือข้อมูลติดตามจาก Supervisor หรือ Admin<br>6. อ่านข้อมูลเหตุผิดปกติเดิมจาก D5 และปรับปรุงสถานะหรือข้อมูลการติดตามใน D5<br>7. บันทึกกิจกรรมการเปลี่ยนแปลงลงใน D6<br>8. ส่งผลการบันทึกหรือผลการอัปเดตกลับไปยังผู้เกี่ยวข้อง |
| Notes       | Staff ทำหน้าที่สร้างข้อมูลเหตุผิดปกติ แต่ไม่อัปเดตสถานะ incident; Supervisor และ Admin เป็นผู้ปรับปรุงสถานะและข้อมูลติดตาม กระบวนการนี้ไม่รวม dashboard/history หรือระบบแจ้งเตือน |

## Process 5.0 : ติดตามภาพรวมและทบทวนประวัติ

| Field       | Detail                                                                    |
| ----------- | ------------------------------------------------------------------------- |
| Process     | 5.0 ติดตามภาพรวมและทบทวนประวัติ |
| Description | กระบวนการสำหรับ Supervisor และ Admin ในการติดตามภาพรวมการทำงาน ตรวจสอบรายการเหตุผิดปกติ ทบทวนประวัติ checklist และ incident activity รวมถึงจัดเตรียมข้อมูลสรุปหรือเอกสารประกอบการทบทวน |
| Input       | คำขอติดตามสถานะงานจาก Supervisor; คำขอดูภาพรวมระบบจาก Admin; ข้อมูลการตรวจเช็กจาก D4; ข้อมูลเหตุผิดปกติจาก D5; ข้อมูลกิจกรรมเหตุผิดปกติจาก D6 |
| Output      | ข้อมูลแดชบอร์ด รายการเหตุผิดปกติ ประวัติ และเอกสารสรุปกลับไปยัง Supervisor; ข้อมูลภาพรวมระบบกลับไปยัง Admin; ข้อมูลสรุปที่ประมวลจาก D4, D5 และ D6 |
| Data Store  | D4 ข้อมูลการตรวจเช็กประจำวัน (`checklist_runs`, `checklist_run_items`); D5 ข้อมูลเหตุผิดปกติ (`incidents`); D6 ข้อมูลกิจกรรมเหตุผิดปกติ (`incident_activities`) |
| Actor       | Supervisor, Admin |
| Task        | 1. รับคำขอดูภาพรวมและสถานะงานจาก Supervisor หรือ Admin<br>2. ส่งเงื่อนไขการดูข้อมูลไปยังการอ่านข้อมูล checklist history จาก D4<br>3. อ่านข้อมูลเหตุผิดปกติจาก D5 เพื่อจัดทำข้อมูลสรุปและรายการเหตุผิดปกติ<br>4. อ่านข้อมูลกิจกรรมเหตุผิดปกติจาก D6 เพื่อใช้เป็นประวัติการดำเนินงาน<br>5. ประมวลผลภาพรวม dashboard จากข้อมูล checklist และ incident<br>6. จัดเตรียมประวัติและเอกสารสรุปที่เกี่ยวข้อง<br>7. ส่งข้อมูลภาพรวม รายการเหตุผิดปกติ ประวัติ และเอกสารสรุปกลับให้ Supervisor หรือ Admin ตามบทบาท |
| Notes       | กระบวนการนี้ไม่รวม Staff และไม่รวม report builder, analytics warehouse, notification, approval workflow, machine registry, external API หรือ AI/copilot |

## Review Notes

- ไม่พบจุดที่ต้องระบุ “ต้องตรวจสอบเพิ่ม” จากข้อมูลที่ใช้ในร่างนี้
- กระบวนการ 1.0 ตั้งใจไม่รวม public signup
- กระบวนการ 2.0 ตั้งใจไม่รวม checklist run execution
- กระบวนการ 3.0 ตั้งใจไม่รวม incident status management
- กระบวนการ 4.0 ตั้งใจไม่รวม dashboard/history และระบบแจ้งเตือน
- กระบวนการ 5.0 ตั้งใจไม่รวม Staff, report builder, analytics warehouse, notification, approval workflow, machine registry, external API และ AI/copilot
